# Architecture Decision Records

This directory contains the project's Architecture Decision Records (ADRs).

Each ADR captures a single architecturally significant decision: the context, the alternatives considered, the choice made, and its consequences.

ADRs are immutable once accepted. If a decision changes, a new ADR is created that supersedes the original.

## Status definitions

- **Proposed** — Under discussion, not yet decided
- **Accepted** — Decision is active and governs work
- **Rejected** — Considered and declined
- **Deprecated** — No longer relevant due to changed context
- **Superseded** — Replaced by a newer ADR (linked in frontmatter)
